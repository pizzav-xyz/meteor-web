# Contributing to Meteor Web

First off, thank you for considering contributing to Meteor Web! It's people like you that make the open-source community such a great place.

## Where to Start

-   **Bug Reports:** If you find a bug, please create an issue and provide as much information as possible, including steps to reproduce the bug.
-   **Feature Requests:** We're open to new ideas! Create an issue to suggest a new feature, and we can discuss it.
-   **Pull Requests:** If you're ready to contribute code, create a pull request. We'll review it as soon as we can.

## How to Contribute

1.  Fork the repository.
2.  Create a new branch for your feature or bug fix:
    ```bash
    git checkout -b my-new-feature
    ```
3.  Make your changes.
4.  Commit your changes:
    ```bash
    git commit -am 'Add some feature'
    ```
5.  Push to the branch:
    ```bash
    git push origin my-new-feature
    ```
6.  Create a new Pull Request.

## Style Guide

We use [Prettier](https://prettier.io/) to format our code. Please run `npm run format` before committing your changes.

We also use [ESLint](https://eslint.org/) to lint our code. Please run `npm run lint` to check for any issues.
